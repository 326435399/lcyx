9.2抽象工厂模式的定义

  1. 定义:
    Provide an interface for creating families of related or dependent objects without specifying their concrete classes.
    翻译:为创建一组相关或互相依赖的对象提供一个接口,而且无须指定他们的具体类.
  2. UML图

    抽象工厂是有多个抽象产品品类,然后抽象的产品类可以共同继承或实现一个抽象类或接口.也就是产品族的划分.
    例子:汽车有左门和右门,不同的汽车门不一样,那么就有了左门族和右门族.
    抽象工厂类就是(提供不同产品族同一品牌的创建方法,例如 法拉利的左门右门,莲花的左门右门,奥迪的左门右门).
    抽象产品类就是(不同的产品族),这里就是左门和右门
    具体产品类就是不同品牌的车门
    客户端不关心具体是什么车门,只知道调用抽象工厂类提供的创建方法,可以创建这个品类(左门/右门)的产品
  3. 代码实现:
